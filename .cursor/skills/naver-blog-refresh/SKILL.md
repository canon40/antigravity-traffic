# 네이버 블로그 새로고침 (Autoblog)

기존 네이버 블로그 글을 **네이버 AI 브리핑·메이트 5원칙**에 맞게 진단·재구성·초안 작성한다.
신규 글 생성(`blog_content_gen`)이 아니라 **이미 발행된 글의 자산을 살려 재발행**하는 워크플로다.

## 프로젝트 컨텍스트 (먼저 읽기)

작업 전 아래를 확인한다. 내용이 충돌하면 wiki·코드가 우선이다.

- `wiki/03_naver_mate.md` — 5원칙 상세
- `wiki/00_core.md` — 7:2:1, 금지어, 최소 1,500자
- `wiki/02_post_types.md` — 글 유형·범위
- `blog_content_gen.py` — `_NAVER_MATE_FIVE_PRINCIPLES`, `_finalize_blog_article` 후처리 규칙

## 입력 요청

사용자에게 없으면 짧게 묻는다:

1. **기존 글 전문** (제목·본문·태그, 이미지 캡션 가능하면 포함)
2. **글 유형** (자동차/바이크/코팅제/제품 홍보/맛집·일상/취미/정보성 팁 등)
3. **계정 말투** — `hymini1`(격식·전문) / `hymini11`(친근) / 미지정이면 원문 말투 유지
4. **상업 관계** — 협찬·체험·내돈내산 여부
5. **추가 경험 메모** — 현장 일화·수치·Before/After (있으면 반드시 반영)

## 5단계 워크플로 (순서 고정)

각 단계 출력 후 **다음 단계로 넘어가기 전** 사용자 확인을 받는다. 한 번에 전부 끝내지 않는다.

### 1단계 — 진단

아래 체크리스트로 **통과/부족/해당없음**을 표시한다.

| 항목 | 점검 내용 |
|------|-----------|
| 직접 경험 | 시공·실패·현장·구체 수치 2문장 이상 |
| 일관 주제 | 글 유형·Truth Table 범위, 잡주제·짜깁기 없음 |
| 진정성 | 협찬·체험·내돈내산 상단 고지 |
| 구조 | `##` 소제목, 짧은 문단, 검색 의도↔제목 일치 |
| 최신성 | 구체 시점(최근 시공·올해 장마 등) |
| 이미지 | 사진 핵심이 본문·표에도 텍스트로 서술됨 |
| 금지 | 취소선, 그래핀=주성분 단정, 상투 인사·맺음말 |
| AI 흔적 | 기계적 요약·경험 없는 나열·중복 소제목 |

**출력**: 진단표 + **치명 3건 이하** 우선 수정 항목 + 수정 vs 재발행 권고(아래 기준).

**수정 vs 재발행**

- 일부 정보만 변경 → 기존 글 **수정** 권고
- 구조·제목·사진 설명·검색 의도 전면 재정비 → **새 글 재발행** 권고
- 기존 글 **즉시 삭제 비권장** (새 글 발행 후 정리는 사용자 판단)

### 2단계 — 구조 재설계

진단 결과를 반영해 **개요만** 작성한다. 본문은 쓰지 않는다.

```markdown
# [새 제목 후보 2~3개] — 검색 의도 + 경험 신호, 판매 문구 금지

## 도입 (2~3문장) — 독자 상황·질문
## [소제목 1] → 한 줄 설명
## [소제목 2] → ...
(총 4~6개 ##)
## 요약·마무리 — 독자 질문 1문장
```

- 흐름: **공감(상황) → 해결(원리·팁·표) → 입증(현장 경험)**
- 7:2:1 — 정보 70% / 전문 20% / 홍보 10%

### 3단계 — 새로고침 초안 (1차)

2단계 개요대로 **전체 본문** 작성.

**반드시 지킬 것**

- 원문의 **경험·수치·일화**는 삭제하지 말고 보강
- 작성자 **말투** 유지 (계정 지정 시 해당 톤)
- 본문 **최소 1,500자** (부족하면 경험·표·FAQ로 보강, 허위 채우기 금지)
- 상업 고지가 필요하면 **맨 앞 한 줄**
- `[IMAGE]` 위치가 필요하면 마커 + `(이미지 설명) … 본문에도 핵심 서술` 힌트
- 마지막 280자 안에 **독자 질문 1문장** (`?`)

**금지어**: 안녕하세요, 알아보겠습니다, 도움이 되었으면 합니다

태그 줄: `태그: 키워드1, 키워드2, …` (5~8개, 글 유형과 일치)

### 4단계 — 경험 인터뷰

초안에서 **비어 있는 경험 슬롯**만 골라 질문한다 (5문 이내).

예시 유형:

- 현장에서 실제로 본 Before/After
- 실패·시행착오 한 가지
- 구체 수치(면적, 시간, 온도, 횟수)
- 독자가 헷갈리는 비교 기준
- 최근(2025~2026) 업데이트된 사실

사용자 답변을 받은 뒤 **해당 문단만** 초안에 반영한다.

### 5단계 — 재발행 패키지

최종 산출물:

```markdown
## 제목 (확정)
## 태그
## 본문 (발행용)
## 작성자가 직접 채울 20~30% 체크리스트
- [ ] 사진·캡션·대체 텍스트
- [ ] 고지 문구 사실 확인
- [ ] 수치·가격·일정 최종 검증
- [ ] 윤문(말투·오타)
- [ ] 재발행 vs 수정 최종 결정

## 운영 메모
- 권장 방식: 수정 / 새 글 재발행
- 기존 글 URL 처리: 삭제하지 말 것
```

**기대치**: AI 초안은 구조화 **70~80%**. 나머지는 사람이 경험·사실·윤문으로 완성한다.

## Autoblog 파이프라인 연동 (선택)

사용자가 코드 연동을 요청할 때만:

- 초안을 `drafts/` 등에 저장
- `experience_notes`·`sponsorship_label` 형태로 config에 넣어 `blog_content_gen` 재사용 검토
- 자동 발행은 **사용자 명시 요청 후**에만

## 트리거 문구 예시
