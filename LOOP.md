## 목표

- (여기에 이번 작업의 한 줄 목표)

## 완료 선언 금지 조건

다음이 **하나라도** 미충족이면 완료로 보고하지 않는다.

### 1. 필수 통과 (Pass/Fail)

- [ ] 빌드 성공
- [ ] 타입 체크 성공
- [ ] 린트 성공
- [ ] 테스트 성공
- [ ] (프로젝트별) 마이그레이션·보안 키 노출 없음

### 2. 설계 대조

- [ ] PRD / 요구사항과 일치
- [ ] 사용자 흐름과 화면·API 일치
- [ ] DB·스키마와 충돌 없음
- [ ] 작업 목록(Tasks) 누락 없음
- [ ] 코딩 컨벤션 준수

### 3. 증거 보고 (완료 시 제출)

- 실행한 명령과 결과 요약
- 변경 파일 목록
- 남은 리스크·사람 확인이 필요한 항목

## 자동 처리 OK

- 린트·타입 오류 단순 수정
- 테스트 실패 로그 기반 단순 버그 수정
- 누락 테스트 추가, 문서 동기화
- 리뷰의 단순 네이밍·포맷 지적

## 반드시 사람에게 질문

- DB 스키마·데이터 손실 가능 마이그레이션
- 인증·권한·결제·보안 정책 변경
- PRD와 충돌하는 리뷰·범위 확대
- 테스트는 통과하나 설계 방향이 바뀌는 변경

## 운영 루프 (선택)

PR/CI 사용 시:

- N분마다 PR·CI 상태 확인
- 단순 실패는 수정 후 재푸시
- 구조·정책 변경 리뷰는 멈추고 질문
